# 📜 사회 분석 대본 프레임워크

## 1. 대본 모델 개요

대본은 **STORM** 모델로 구축합니다 — 전통적 문제-해결 모델의 강화 버전:

```
S — SHOCK (충격)      → Hook 오프닝, 역설, 의외의 수치
T — THINK (사고)      → 문제 전개, 심층 분석
O — OBSERVE (관찰)    → 데이터, 증거, 실제 사례 연구
R — REFLECT (반영)    → 시청자 개인과 연결, 참여 유도
M — MOVE (행동)       → 실용적 해결책, 행동 촉구
```

---

## 2. 상세 대본 구조 (6개 파트)

### 파트 0: 대본 메타데이터 (작성 전)

대본 작성에 앞서 다음 정보를 확정:

```yaml
title: "[영상 제목]"
topic_pillar: "경제 | 사회심리 | 시스템구조 | 전망전략"
target_length: "12-20분"
target_word_count: "2400-4000자"
core_paradox: "[영상의 핵심 역설]"
data_sources:
  - "[출처 1]"
  - "[출처 2]"
  - "[출처 3]"
key_metaphors:
  - "[은유 1]"
  - "[은유 2]"
emotional_arc: "충격 → 불안 → 분노 → 공감 → 각성"
cta_type: "댓글 | 공유 | 구독 | 전체"
```

### 파트 1: THE HOOK — 오프닝 펀치 (0:00 – 0:45)

**목적**: 처음 5초 안에 시청자가 떠나지 못하게 만든다.

**구조**:

```
[충격적 첫 문장, 1문장, 짧고 무겁게]

[신뢰 출처 기반 수치]

[역설, "왜 A인데 B인가?"]

[약속, "이 영상에서 X개의 폭풍을 해독합니다..."]

[티저, "세 번째 폭풍은 모든 것을 다시 보게 만들 겁니다."]

[면책고지(배경청취): 배경 청취 안내 + 출처 정보]
```

**예시**:

> *"2025년, 한국의 1인당 GDP가 4만 3천 달러를 돌파했습니다. 하지만 아무도 크게 말하지 않는 진실이 있습니다: 30세 미만 청년의 72퍼센트가 현재 일자리가 있는데도 재정적 탈진 상태에 있습니다.*
>
> *한국은행에 따르면, 지난 10년간 1인당 소득은 40퍼센트 증가했습니다. 하지만 평균 내 집 마련 가능성은 60퍼센트 감소했습니다.*
>
> *그렇다면 돈은 어디로 갔나요? 누가 이득을 보고 있나요? 그리고 역사상 가장 운이 좋다고 불리는 세대가 왜 가장 지쳐있나요?*
>
> *오늘, 한국 청년 세대를 갉아먹고 있는 3개의 보이지 않는 폭풍을 해독합니다. 세 번째 폭풍은 대부분의 사람들이 존재조차 인식하지 못한 것입니다.*
>
> *본격적 내용에 앞서, 이 영상은 집안일이나 요리를 하면서 배경으로 듣기에 좋습니다. 영상 속 정보는 여러 출처에서 수집 정리한 것입니다. 틀린 부분이 있다면 댓글로 남겨주세요."*

**기술 참고사항**:
- 첫 문장은 반드시 **수치 또는 충격적 선언** 포함
- Hook에서 인사말이나 채널 소개 없음
- 티저 후 반드시 **면책고지** (배경 청취 안내 + 출처 수집 관련)

---

### 파트 2: CONTEXT BRIDGE — 맥락 연결 (0:45 – 2:00)

**목적**: 시청자가 이 문제가 왜 자신에게 중요한지 이해할 수 있는 큰 맥락 제공.

**구조**:

```
[큰 그림 — 세계/한국 배경 설정]

[개인 연결 — "이게 익숙하게 느껴진다면..."]

[자체 용어 정의 — 현상에 이름 붙이기]

[안내 — "X개 파트를 거칠 것이며, 각 파트는 한 층 더 깊어집니다"]
```

---

### 파트 3: DEEP ANALYSIS — 심층 분석 (2:00 – 14:00)

**목적**: 3-4개의 "폭풍"을 데이터, 은유, 참여 질문으로 전개.

**각 "폭풍"은 DARE 구조**:

```
D — DATA (데이터)     → 신뢰 출처의 구체적 수치
A — ANALOGY (은유)    → 수치를 생생한 이미지로 전환
R — REALITY (현실)    → 사례 연구, 실화, 일상 예시
E — ENGAGE (참여)     → 시청자 질문, 댓글 유도
```

**참고사항**:
- 각 폭풍은 **3-5분**
- 폭풍 사이에 **전환 문장**: *"하지만 여기까지만 보면 일부만 본 겁니다..."*
- 마지막 폭풍이 **클라이맥스** — 가장 충격적 정보 또는 의외의 관점
- **수사학적 질문**을 삽입하여 리듬 유지

---

### 파트 4: THE MIRROR — 반영 & 공감 (14:00 – 16:00)

**목적**: 시청자의 감정을 확인하고 깊은 연결 형성.

**구조**:

```
[감정 확인 — "여기까지 듣고 무겁다면..."]

[정상화 — "완전히 자연스러운 반응입니다."]

[재확인 — "피로는 여러분이 약해서가 아닙니다."]

[방향 전환 — "하지만 문제를 알고 행동하지 않으면 소용없습니다."]
```

---

### 파트 5: THE BLUEPRINT — 해결책 & 전략 (16:00 – 18:30)

**목적**: **3가지 구체적, 실용적, 즉시 시작 가능한 행동 전략** 제시.

**구조**:

```
[기본 철학 — 모든 전략의 기초가 되는 원칙]

[전략 1 — 구체적, 실용적, 오늘 당장]
  ↳ 왜 효과가 있는지
  ↳ 시작 방법 (1-2 첫 단계)

[전략 2 — 중기적, 기반 구축]
  ↳ 왜 효과가 있는지
  ↳ 시작 방법

[전략 3 — 사고 전환, 장기적]
  ↳ 왜 효과가 있는지
  ↳ 시작 방법

[강력한 확언 — "진짜 무기는..."]
```

---

### 파트 6: THE CLOSE — 결론 & CTA (18:30 – 20:00)

**목적**: 강력한 선언과 명확한 행동 촉구로 마무리.

**구조**:

```
[1문장 요약, 처음 역설 + 해결책]

[마무리 철학, 인용할 만한 1문장]

[CTA 1, 댓글: 구체적 질문]

[CTA 2, 공유: "이게 필요한 누군가를 알고 있다면..."]

[CTA 3, 구독 + 알림]

[면책고지 마무리: 출처 수집 안내 + 수정 요청]

[마지막 문장, 짧고 강하고 기억에 남게]
```

## 3. 텍스트 전용 대본 작성 가이드

### 핵심 원칙

대본 출력물은 **텍스트만** 포함하며, 기술 기호, 마크다운, 주석은 없습니다. 읽는 사람(또는 TTS 엔진)이 어떤 처리 없이 바로 읽을 수 있어야 합니다.

### TTS 친화 규칙 (필수)

대본은 사람 또는 AI TTS로 **자연스럽게 읽을 수 있어야** 합니다.

| ❌ 사용 금지 | ✅ 이렇게 적기 | 이유 |
|-------------|---------------|------|
| `GDP` | 국내총생산 | TTS 오독 |
| `$` | 달러 | TTS 기호 오독 |
| `₩` | 원 | TTS 기호 오독 |
| `50%` | 50퍼센트 | TTS % 오독 |
| `1.5만` | 1만 5천 | TTS 소수점 오독 |
| `200,000` | 20만 | TTS 쉼표 오독 |
| `3-4배` | 3배에서 4배 | TTS 하이픈을 '빼기'로 읽음 |
| `—` (대시) | `,` 또는 `.` | TTS 대시 미인식 |
| `통계청` | 통계청 | 약어 OK (한국어 기관명은 그대로 사용 가능) |
| `IMF` | 국제통화기금 | 영어 약어 풀어쓰기 |
| `ILO` | 국제노동기구 | 영어 약어 풀어쓰기 |

**황금률**: 모든 문장을 소리내어 읽으세요. 읽는 법을 설명해야 한다면, 다시 써야 합니다.

### 서식 규칙

| 요소 | 처리 방법 |
|------|-----------|
| **단락 구분** | 큰 아이디어 사이에 빈 줄 |
| **강조** | 자연스러운 문장 구조로 강조 |
| **쉼** | 짧은 문장을 한 줄에 분리 |
| **긴장 고조** | 연속 질문 3개 |
| **파트 전환** | 빈 줄 2개 |
| **문장부호** | 마침표, 쉼표, 물음표, 콜론만 사용 |

### 대본 필수 포함 단락

**1. 배경청취 면책고지 (Hook 이후, Bridge 전):**
> *"본격적 내용에 앞서, 이 영상은 집안일이나 요리를 하면서 배경으로 듣기에 좋습니다. 영상 속 정보는 여러 출처에서 수집 정리한 것입니다. 틀린 부분이 있다면 댓글로 남겨주세요."*

**2. 마무리 면책고지 (마지막 문장 전):**
> *"다시 한번 말씀드립니다. 이 영상의 정보는 여러 출처에서 수집 정리한 것입니다. 잘못된 부분이나 보완이 필요한 부분이 있으면 댓글로 알려주세요."*

### 텍스트를 활용한 리듬 기법

1. **짧은 문장 = 침묵**: 3-5자 문장을 별도 줄에 놓으면 자연스러운 쉼 생성
2. **연속 질문 = 긴장 고조**: 3개 질문 연속이 밀도감 창출
3. **구조 반복 = 강조**: "노력이 부족해서가 아닙니다. 실력이 없어서가 아닙니다. 시스템이..."
4. **1문장 단락 = 드라마**: 중요 문장 하나를 독립 단락으로 분리
5. **긴 문장+짧은 문장 = 펀치**: 설명 후 짧은 충격 문장으로 마무리

## 5. 대본 작성 프로세스 (단계별)

1. **주제 선정** → 핵심 역설 확정
2. **데이터 수집** → 최소 5개 출처, 가장 강한 수치 3개 선택
3. **3개 폭풍 확정** → 충격 강도 오름차순 배열
4. **Hook 먼저 작성** → 3개 버전 작성, 가장 강한 것 선택
5. **Deep Analysis 작성** → 각 폭풍에 DARE 구조 적용
6. **Mirror + Blueprint 작성** → 해결책이 구체적이고 실용적인지 확인
7. **Close 작성** → 명확한 CTA, 기억에 남는 마무리
8. **Bridge 작성** → 마지막에 (전체 내용을 알아야 좋은 연결 가능)
9. **검토 & 다듬기** → 전체를 소리내어 읽기, 불필요 문장 삭제, 체크리스트 확인, TTS 호환 확인
10. **면책고지 2개 확인** → Hook 후 배경청취 면책고지 + 마무리 면책고지
11. **txt 파일 2개 출력** → 텍스트 전용 대본 + 제목 및 에셋 파일

## 6. 언어 적응 규칙 (다국어)

한국어 대본 핵심 적응 사항:

| 요소 | 한국어 (ko) |
|------|-------------|
| **호칭** | 여러분 / 우리 |
| **톤** | 격식+친근 존댓말 |
| **은유** | K-문화 (경쟁, 사교육, 아파트, 치킨+맥주) |
| **수치 출처** | 통계청, 한국은행, 한국보건사회연구원 |
| **CTA** | 댓글, 공유, 구독 |
| **해시태그** | 한국어 + 영어 |
| **썸네일** | 서울, 아파트 단지, 강남 등 한국 배경 |

### 적응 원칙

1. **번역이 아닌 재작성** — 목표 언어로 자연스럽게 완전히 다시 씀
2. **은유는 문화에 맞게** — 원본 은유가 의미 없으면 완전 교체
3. **수치는 현지 출처** — 해당 국가 데이터 사용
4. **STORM + DARE 구조는 유지** — 프레임워크는 보편적, 내용만 교체
5. **썸네일은 현지 배경** — 시청자가 "이건 내 이야기"라고 느껴야 함
